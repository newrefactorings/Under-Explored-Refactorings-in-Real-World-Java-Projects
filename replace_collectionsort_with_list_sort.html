Refactoring Definition:
Replace Collections.sort with List.sort involves replacing the static Collections.sort method with the more concise List.sort method to sort a list in place using a comparator.

Rationale:

  • Improved Readability: Using List.sort directly is more concise and aligns with modern Java practices, reducing boilerplate code.
  • Increased Clarity: The List.sort method explicitly operates on the list instance, making the code more intuitive and object-oriented.
  • Modern Java Best Practices: List.sort was introduced in Java 8, promoting the use of instance methods over static utility methods where appropriate.
  • Consistency: Replacing Collections.sort with List.sort ensures consistency with other instance methods, improving maintainability.
