This website presents a curated collection of under-explored refactorings observed in real-world open-source Java projects. Unlike widely studied transformations, these refactorings often go unnoticed in research and tooling, yet they offer practical value for improving code structure and maintainability. The listed refactorings fall into two categories: traditionally adaptable refactorings and context-specific refactorings.

Traditionally adaptable refactorings are those that can be applied across diverse systems without requiring detailed knowledge of the surrounding application or domain-specific constraints.

Context-specific refactorings are those applied to address recurring patterns, constraints, or conventions within a particular application domain, framework, or project context. These refactorings depend heavily on local implementation details or usage patterns and are marked with a superscript **.
