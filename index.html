Replace Refactorings

Replace refactorings involve substituting one part of the code with another to improve its structure, readability, or performance while preserving its functionality. The emerging refactorings in this edition of refactorings are presented in the following list.

  • Replace ‘expected’ attribute with try-catch
  • Replace array with list
  • Replace array with singleton list
  • Replace array list with primitive array
  • Replace collections.sort() with list.sort()
  • Replace customized methods with API
  • Replace for with while
  • Replace HashMap with linkedHashMap
  • Replace inner class with lambda expression
  • Replace primitive with object
  • Replace wrapper object with primitive
  • Replace iteration-based retrieval with switch expression

Modification Refactorings

Modification refactorings change the existing code to improve its design, clarity, or efficiency without adding new functionality or removing existing functionality.The emerging refactorings in this edition of refactorings are presented in the following list.

  • Turn local variable into field
  • Increase field visibility
  • Loosen parameter’s type limitations
  • Avoid unnecessary actions by precondition checking
  • Combine enum with comparator
  • Convert static nested class to inner class
  • Decrease field visibility
  • Simplify source code by reusing existing data type
  • Parameterize links

Addition Refactorings

Addition refactorings involve adding new functionality to the existing codebase. This might include introducing new classes, methods, or components to enhance the system without altering its existing behavior. The goal is to extend the system's capabilities in a way that is maintainable and follows good design principles. The emerging refactorings in this edition of refactorings are presented in the following list.

  • Add brackets
  • Add delegation for super class’s constructor

Remove Refactorings

Remove refactorings eliminate unnecessary or redundant parts of the code, such as unused variables, dead code, or obsolete methods, to simplify and clean up the codebase.The emerging refactorings in this edition of refactorings are presented in the following list

  • Remove debug logging notes
  • Remove brackets
  • Remove duplicate condition checks
  • Remove duplication in constructor by delegating to another constructor
  • Remove overriding method that equals to its overridden method
  • Remove repetitive checking
  • Remove suppress warning annotation
  • Remove unnecessary array creation
  • Remove unnecessary compilation dependency
  • Remove unnecessary extends clause
  • Remove unnecessary type cast
