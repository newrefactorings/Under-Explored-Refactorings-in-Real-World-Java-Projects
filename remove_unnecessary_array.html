Refactoring Definition:
Remove Unnecessary Array Creation involves replacing the creation of an array when a single element is sufficient, by directly passing the element to the method or constructor.

Rationale:

  • Improves Code Simplicity: Reduces unnecessary complexity in the code by avoiding redundant array creation.
  • Readability: Simplifies the method call or constructor invocation, making the code easier to understand.

Note: This is a context-specific refactoring applied when the array creation is no longer necessary due to a change in method signature or when the element passed is singular and does not require an array.
